Commande ls et caractères génériques
Dans le dossier /usr/bin...
Remarques : dans certains shell le caractère de négation est soit ! soit ^
u = user g = groupe o = others ? = pour dire un caractère quelconque à une position donnée
~ = chemin dossier personnel 

r = read
w = write
x = execute
en octal 7 = tous les droits cad rwx pour le reste c'est juste en focntion de la position 
convertir en décimal exemple r-x : position de x à 0 et r à 2 donc : 2⁰ + 2² = 5

Question 1 : Listez les fichiers dont le nom est formé par exactement quatre lettres.
reponse : ls -d [a-zA-Z][a-zA-Z][a-zA-Z][a-zA-Z]

Question 2 : Listez les fichiers dont le nom est formé par au-moins quatre lettres.
reponse : ls -d [a-zA-Z][a-zA-Z][a-zA-Z][a-zA-Z]* # * n'importe quoi 

Question 3 : Listez les fichiers dont le nom contient le caractère ‘-‘.
reponse : ls -d *-*

Question 4 : Listez les fichiers dont le nom contient le caractère ‘-‘ placé en 3ème position.
reponse : ls -d ??-*

Question 5 : Listez les fichiers dont le nom commence par la lettre a, b ou c.
reponse : ls -d [abc]*

Question 6 : Listez les fichiers dont le nom ne commence pas par la lettre a, e, i, o ou u.
reponse : ls -d [^aeiou] *

Question 7 : Listez les fichiers dont le nom ne commence pas par une lettre comprise entre c et v.
reponse : ls -d [^c-v]*

Gestion des droits d’accès
Pré requis : Créez un fichier file3.sh dans votre répertoire TP1, qui contient :
#!/bin/bash
echo Mon premier script !

Question 1 : Quels sont les droits d’accès de ce fichier ? 
reponse : -rw-r--r-- 1 sita sita 38 29 nov.  18:52 file3.sh # j'ai le droit de lire et ecrire, les autres que de lire

Exécutez ce fichier avec ./file3.sh. Que se passe-t-il ? 
reponse : bash: ./file3.sh: Permission non accordée

Changez vos droits pour rendre ce fichier exécutable. Exécutez à nouveau ce fichier. Que se
passe-t-il ?
reponse : chmod 700 file3.sh # 7 me donne tout les dorits (droit user), 0 n'accorde rien à mon groupe , 0 n'accorde rien aux autres
le programme affiche : Mon premier script !

Question 2 : Quelle commande permet d’afficher les droits de votre répertoire personnel ? Votre
commande ne doit afficher qu’une seule ligne.
reponse : ls -ld ~

Question 3 : Modifiez les droits de votre répertoire personnel en 700. 
reponse : chmod 700 ~

Votre binôme arrive-t-il à afficher la liste des fichiers de votre répertoire personnel ? Votre binôme arrive-t-il à exécuter le
contenu de votre fichier file3.sh ?
reponse : normalement non, mais pas testé

Question 4 : Ajoutez le droit d’exécution à votre répertoire personnel pour ceux de votre groupe et les
autres. Votre commande doit utilisation la notation symbolique.
reponse : chmod g+rwx,o+rwx ~ # + : add 

Question 5 : Votre binôme arrive-t-il à lire maintenant le contenu de votre répertoire personnel ?
Votre binôme arrive-t-il maintenant à exécuter le contenu de votre fichier file3.sh ?
reponse : normalement il doit pouvoir, mais pas testé

Question 6 : Remettez les droits initiaux de votre répertoire personnel (ceux de la question 2) en
utilisant la notation en octal.
reponse : droits initaux : drwx------ #d pour directory
donc chmod 700 ~

Question 7 : Affichez le masque courant. 
reponse : umask 
On obtient : 0022 # les 3 derniers chiffres nous interessent dans la determination des droits
#rw-r--r-- pour les fichiers et rwxr-xr-x pour les répertoires

Modifiez ce masque pour que les droits à la création soient pour les fichiers rw- rw- --- et pour les répertoires rwx rwx --x. Vérifiez que votre nouveau masque
fonctionne correctement.
reponse : rw- rw- --- correspond à 660 donc en umask à 006 # a revoir cette question


Redirection des entrées/sorties
Avant de commencer, visualisez la liste des utilisateurs connectés en utilisant la commande who.

Question 1 : Insérez le résultat de la commande who dans un fichier file5.txt. Visualisez le contenu du
fichier file5.txt.
reponse : who > file5.txt # le > signifie redirige sortie vers
        cat file5.txt #p cat pour afficher le contenu 

Question 2 : Tapez la commande pwd > file5.txt. Visualisez le contenu du fichier file5.txt. Que
remarquez-vous ?
reponse : cat file5.txt # affiche /home/sita/esisar/tps/os/resolution/tp1

Question 3 : Tapez la commande ls >> file5.txt. Visualisez le contenu du fichier file5.txt. Que
remarquez-vous ?
reponse : la commande affiche 
/home/sita/esisar/tps/os/resolution/tp1
file3.sh
file5.txt
tp1.txt

donc le > écrase le fichier par la nouvelle sortie et >> ajoute justye la nouvelle sortie

Question 4 : Tapez la commande ls -l > file5.txt. Que remarquez-vous ? Même question avec la
commande ls -l 2> file5.txt.

reponse : ls -l > file5.txt # affiche 
total 12
-rwx------ 1 sita sita   38 29 nov.  18:52 file3.sh
-rw-r--r-- 1 sita sita    0 29 nov.  19:46 file5.txt
-rw-r--r-- 1 sita sita 5171 29 nov.  19:46 tp1.txt

ls -l 2> file5.txt # le chiffre 2 rrepresnete la sortie des erreurs : stderr. La commande affiche 
total 12
-rwx------ 1 sita sita   38 29 nov.  18:52 file3.sh
-rw-r--r-- 1 sita sita    0 29 nov.  19:47 file5.txt
-rw-r--r-- 1 sita sita 5357 29 nov.  19:46 tp1.txt

Question 5 : Pour terminer cet exercice, nous allons lister le contenu d'un dossier qui n'existe pas.
Pour chacune des trois commandes suivantes, expliquez ce qui est affiché à l'écran (et pourquoi !) et
ce que contient le fichier erreur.txt après l'exécution de chacune des commandes.
Question 5a : ls /n/existe/pas > erreur.txt
reponse : ls: impossible d'accéder à '/n/existe/pas': Aucun fichier ou dossier de ce type
# c'est normal fichier inextistant donc rien dans stdout

Question 5b : ls /n/existe/pas 2> erreur.txt
reponse : le message ls: impossible d'accéder à '/n/existe/pas': Aucun fichier ou dossier de ce type
est redirigé dans le fichier texte erreur.txt à cause du 2 de la commande

Question 5c : ls /n/existe/pas &> erreur.txt

est redirigé dans le fichier texte erreur.txt à cause du & de la commande : qui signifie sortie stdout et stderr
