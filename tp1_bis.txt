Liens physiques et liens symboliques

remarques :     
    Lien Symbolique (ou lien mou) :
        Un lien symbolique est un fichier distinct qui contient une référence au chemin d'un autre fichier ou répertoire.
        Il crée une association indirecte entre le lien symbolique et la cible.
        Si la cible est supprimée ou déplacée, le lien symbolique peut devenir "cassé" (pointe vers un emplacement invalide).
        Création : ln -s cible lien_symbolique

    Lien Physique (ou lien dur) :
        Un lien physique est une autre entrée de répertoire qui pointe vers le même contenu de disque que l'original.
        Il crée une association directe entre le lien physique et la cible.
        Si la cible est supprimée, le lien physique continue de pointer vers le contenu tant que d'autres liens physiques existent.
        Création : ln cible lien_physique

Commande find

Question 1 : Recherchez, en une seule commande, tous les fichiers qui contiennent dans leur nom la
chaîne de caractères « gre » dans les répertoires /bin et /usr/bin.
reponse : find /bin /usr/bin -type f -name "*gre*" # type f = type fichier 


Question 2 : Recherchez dans votre dossier personnel tous les fichiers qui ont été modifiés depuis
moins de 2 jours.
reponse: find ~/ -type f -mtime -2


Question 3 : Recherchez, dans les répertoires /bin et /usr/bin, en une seule commande, tous les
fichiers réguliers exécutables par tous et affichez les informations relatives à ceux-ci sous cette forme :
N° d’inoeud
droit en octal
nom du fichier avec son chemin
reponse:
find /bin /usr/bin -type f -executable -exec stat -c "%i %a %n" {} \; #pour chaque fichier exécuté on met en place

250ko.

Commande grep
Pour cet exercice, toutes les recherches vont se faire dans les fichiers .h du répertoire /usr/include.
Question 1 : Avec la commande grep -v, recherchez toutes les lignes qui ne contiennent pas la chaîne
de caractères « printf ».
reponse : grep -v 'printf' /usr/include/*.h
Explications :
    grep -v 'printf': Recherche toutes les lignes qui ne contiennent pas la chaîne "printf".
    /usr/include/*.h: Spécifie les fichiers .h dans le répertoire /usr/include.

Question 2 : Recherchez toutes les lignes qui contiennent une succession de trois caractères
identiques compris entre a et e.
reponse : grep -E '(a{3,}|b{3,}|c{3,}|d{3,}|e{3,})' /usr/include/*.h
Explications :
    grep -E: Active les expressions régulières étendues.
    (a{3,}|b{3,}|c{3,}|d{3,}|e{3,}): Recherche une succession d'au moins trois caractères identiques parmi a, b, c, d, ou e.
    /usr/include/*.h: Spécifie les fichiers .h dans le répertoire /usr/include.
